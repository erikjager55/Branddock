# Probleem

Layer 3 in de 3-laagse architectuur (per plan §2): item-specific regression-cases uit productie-feedback. Bronnen: `LearningEvent type=content.rejected` (hard-negative), `LearningEvent type=content.edited` met `editDistance > 0.20` (soft-negative), `BrandReviewFinding severity=CRITICAL` (regression-seeds), manual "report bad output" button. Auto-promote naar Layer 2 wanneer pattern verschijnt ≥3× over verschillende types.

# Voorstel

**LearningEvent → regression-corpus pipeline**:
1. Query alle `content.rejected` + `content.edited` (editDistance > 0.20) van laatste 30 dagen
2. Per item: extract input (brief), output (rejected/edited variant), expected (preferred-variant), reason (finding-categories)
3. Auto-add naar `tests/content-regression-corpus/<auto-generated>.yaml` (Promptfoo-format)
4. Tag met `source: learning-event`, `severity: hard-negative | soft-negative`, `pattern-cluster: <auto-detected>`
5. Nightly run tegen latest prompt-version + alert bij regression > 5% baseline

**Auto-promote logic**:
- Pattern detection: TF-IDF op finding-categories + content-types
- Threshold: ≥ 3 instances + ≥ 2 verschillende content-types = promote naar Layer 2 golden-set
- Manual override: "promote this to regression suite" button in Studio UI
