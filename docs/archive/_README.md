# archive/

Documenten die niet meer in actieve scope zijn maar bewaard worden voor historische context.

**Niet weggooien** — vaak waardevol voor "hoe deden we X 6 maanden geleden?" of audit-trails.

## Subdirectories

| Directory | Inhoud | Voorbeeld |
|---|---|---|
| `implementatieplannen/` | Afgeronde IMPLEMENTATIEPLAN-*.md bestanden | `IMPLEMENTATIEPLAN-BRAND-VOICE.md` |
| `handovers/` | Sessie-overdracht documenten | `HANDOVER-17-FEB-2026.md` |
| `plans-superseded/` | Plannen die vervangen zijn door betere implementaties | `IMPLEMENTATIEPLAN-MULTI-AGENT-STRATEGY.md` (vervangen door CQP) |
| `old-lists/` | Oude TODO/strategische lijsten | `TODO.md` (oude versie), `STRATEGISCHE-VERVOLGSTAPPEN.md` |

## Beleid

- **Geen edits** in archieven — alleen toevoegen of verwijderen
- **Verplaatsing alleen na user-goedkeuring** (week 1 dag 3)
- Voor referenties vanuit code: gebruik permalink naar git-commit, niet naar pad in archive
